# YAML Schema 规范文档

本文档定义了 `01_source/` 目录下YAML文件的规范结构。所有文件必须严格按照此规范填写。

---

## 1. core_setting.yaml - 核心设定

### 顶层结构

```yaml
# 必须包含以下顶级字段
世界观: <string>           # 必填：世界观背景描述
魔力体系: <string>         # 可选：力量/魔法体系说明
核心冲突: <string>         # 必填：主线矛盾与冲突
人物小传: <dict>           # 必填：角色设定字典
伏笔清单: <list[string]>   # 可选：伏笔列表
设定禁忌: <list[string]>   # 可选：写作禁忌事项
风格约束: <dict>           # 可选：风格相关约束
```

### 详细结构

#### 1.1 世界观 (必填)
```yaml
世界观: |
  多行字符串，描述故事背景、世界规则、社会结构等。
  支持换行，使用 | 符号保留格式。
```

#### 1.2 魔力体系 (可选)
```yaml
魔力体系: |
  描述力量体系、等级划分、特殊规则等。
```

#### 1.3 核心冲突 (必填)
```yaml
核心冲突: |
  描述主线矛盾、主角的核心目标与障碍。
  可以包含多个冲突层次。
```

#### 1.4 人物小传 (必填)
```yaml
人物小传:
  角色名1:
    角色类型: <string>      # 主角/主要配角/次要配角/反派/龙套
    身份: <string>          # 角色身份
    性格: <string>          # 性格特点
    核心动机: <string>      # 行动动机
    外貌: <string>          # 可选：外貌描述
    能力: <string>          # 可选：能力/技能
    背景故事: <string>      # 可选：过往经历
    # 可以添加更多自定义字段
  
  角色名2:
    角色类型: 主要配角
    身份: ...
    # ...
```

#### 1.5 伏笔清单 (可选)
```yaml
伏笔清单:
  - "伏笔1描述"
  - "伏笔2描述"
  - "..."
```

#### 1.6 设定禁忌 (可选)
```yaml
设定禁忌:
  - "禁忌1：不得出现XXX"
  - "禁忌2：角色不得做出XXX行为"
```

#### 1.7 风格约束 (可选)
```yaml
风格约束:
  语言风格: <string>        # 如：冷峻、幽默、诗意
  对话特征: <string>        # 对话风格说明
  场景描写: <string>        # 场景描写要求
  节奏控制: <string>        # 叙事节奏要求
```

## 2. overall_outline.yaml - 整体大纲

### 顶层结构

```yaml
# 必须包含以下顶级字段
总章节数: <int>             # 预计总章节数
故事概述: <string>          # 故事整体概要
幕结构: <dict>              # 必填：幕结构字典
关键转折点: <list[dict]>    # 必填：关键转折点列表
```

### 详细结构

#### 2.1 总章节数
```yaml
总章节数: 100
```

#### 2.2 故事概述
```yaml
故事概述: |
  故事整体概要，描述从开端到结局的主线发展。
```

#### 2.3 幕结构 (必填)
```yaml
幕结构:
  第一幕:
    章节范围: "第1-20章"
    概述: "本幕的主要内容和目标"
    剧情要点:
      - "第1-5章：开篇事件"
      - "第6-10章：发展"
      - "第11-20章：第一幕高潮"
  
  第二幕:
    章节范围: "第21-70章"
    概述: "本幕的主要内容和目标"
    剧情要点:
      - "第21-35章：..."
      - "..."
  
  第三幕:
    章节范围: "第71-100章"
    概述: "本幕的主要内容和目标"
    剧情要点:
      - "..."
```

#### 2.4 关键转折点 (必填)
```yaml
关键转折点:
  - 章节: 10
    事件: "主角觉醒能力"
    影响: "改变故事走向"
  
  - 章节: 50
    事件: "重大揭秘"
    影响: "反转剧情"
  
  - 章节: 90
    事件: "最终决战"
    影响: "故事结局"
```

## 3. 数据类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| `<string>` | 单行或多行文本 | `世界观: 这是描述` |
| `<int>` | 整数 | `总章节数: 100` |
| `<list[string]>` | 字符串列表 | `["项目1", "项目2"]` |
| `<list[dict]>` | 字典列表 | `[{章节: 1, 事件: "xx"}]` |
| `<dict>` | 键值对字典 | `{字段1: 值1, 字段2: 值2}` |

---

## 4. 验证规则

1. **必填字段**：标注"必填"的字段必须存在且非空
2. **字段类型**：字段值必须符合声明的类型
3. **嵌套结构**：嵌套字典的子字段必须符合定义
4. **列表格式**：列表项使用 `- ` 前缀
5. **多行文本**：使用 `|` 保留换行格式
